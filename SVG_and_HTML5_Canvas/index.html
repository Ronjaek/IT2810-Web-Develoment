Documentation

Implementation of Flexbox-layout

Flexbox-layout is used in my code to structure the two different artworks side by side if the screen is large enough, and under each other if the screen is too small. The two artworks are placed inside each own flexbox-element. The code for this is wirtten in the following two CSS classes: "flexbox" and "flexbox-element". I have also used CSS @media Rule to change the flex-direction attribute on screens smaller than 1240px, making the two artworks stack vertically in stead of horizontally.

Implementation of HTML canvas and SVG

The artwork that appears to the left if the screen is big enough and on top if you are using a smaller screen is made by SVG elements. The SVG elements are implemented directly throughout the HTML code. The elements in the artwork are a combination of circles, rectangles, lines, curves and text. To create a new element I used either the rect(), ellipse(), line(), path() or text() functions and then specified the coordinates, radius or height/width, stroke, stroke-width and then the fill color. By using the transform attribute I can rotate the elements I have made. To make it easier to find the correct positions for the elements and the correct rotation coordinates I used a SVG editor to illustrate the outcome, before hardcoding it in HTML myself. To make the cat meow and show its tongue, which is part of the interactive part of the artwork, the display of both the following IDs: "svg_speech_bubble" and "svg_cat_tongue" is set to "none" by default in the stylesheet. When the user then clicks anywhere on the artwork jQuery is used to show and hide the cat´s tongue, make the speech bubble fade in and out, as well as make the cat meow. I have given all the elements in the artwork an ID, so I can easily change the attributes of the elements. By changing an element´s coordinates I made it possible to make the cat fatter while hovering over "section1".

The artwork that appears either to the right or on the bottom of the website is made by HTML canvas. While the SVG elements are mainly implemented in the HTML code, HTML canvas are mainly implemented in the JavaScript file. HTML is only used to specify the size of the canvas, its background color and its border. To create the rectangles, circles and lines in the artwork I have used the rect(), arc() and lineTo() functions before finishing with the stroke() and fill() functions. I have separated the different groups of elements in distinct functions. This makes it easier to for example write the cat´s tongue and the speech bubble and then overwrite it with a combination of the clearRect() function and a new drawing to make it look just like the interactive part when the user click on the artwork made by SVG elements. To make the cat fatter when hovering over "section2", I once again used the clearRect() function before switching between writing the fat- and the original cat.

The use of jQuery

jQuery is used to access elements from the HTML code, primarly by using an element´s ID. To initialize all the click()- and hover() functions I used the $(document).ready() function. In addition to make the cat meow and show its tongue in both of the artworks, the click() functions are used to open and close both the modal and the footer. To make the hover() function work optimally combined with the click() function I used a global state variable to keep track of wether the user is hovering or not. As I mentioned earlier I have also used jQuery to draw all the elements in the artwork made by HTML canvas.

Cross browser testing

I have performed cross browser testing manually. By opening the HTML path in different popular browsers, I made sure everything is compatible for these three popular browsers:

  • Google Chrome (version 84.0.4147.135)
  • Microsoft Edge (version: 84.0.522.63)
  • Safari (version: 13.1.2)
